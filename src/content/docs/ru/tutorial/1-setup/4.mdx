В этом учебнике мы будем использовать Github для хранения нашего репозитория и подключения к веб-хосту. Вы можете использовать любой онлайн-провайдер git по своему выбору.

:::note
Если вы уже знакомы с git и имеете свой рабочий процесс, то создайте новый репозиторий Github для своего проекта, используя предпочитаемый вами метод. Перейдите к следующей странице: ["Размещение вашего сайта в Интернете"](/ru/tutorial/1-setup/5/).
:::

## Создание репозитория на GitHub

Хотя есть несколько способов, чтобы сохранить ваш локальный код в Github, в этом учебнике мы рассмотрим метод, который не требует использования git в командной строке.

1. Войдите на сайт GitHub.com в браузере и нажмите <kbd>+</kbd> в верхнем правом углу экрана, чтобы создать новый репозиторий.

2. Выберите имя для своего репозитория. Это имя не обязательно должно совпадать с именем папки вашего проекта.

3. Вам будут предложены варианты, но вы не должны менять никакие из них. Пролистайте вниз и нажмите кнопку <kbd>Создать репозиторий</kbd>.

4. Вам будут предложены различные дальнейшие настройки, но вы не должны использовать их. Запомните URL-адрес вашего репозитория. Теперь вы можете закрыть эту страницу, ничего не делая.

## Отправьте свой локальный код на GitHub

На предыдущем шаге вы внесли изменения в содержимое своей страницы. Это означает, что файлы вашего проекта были изменены, и VS Code должен показать число над значком меню «Источник». В этой вкладке источников вы регулярно будете обновлять свои файлы на Github.

1. Нажмите вкладку «Система управления версиями» в VS Code, чтобы увидеть список файлов, которые изменились. Если вы видите сообщение о том, что вам нужно установить `git`, следуйте предоставленным инструкциям, а затем перезагрузите VS Code.

2. Нажмите на меню <kbd>•••</kbd> «3 точки» над сообщением об изменении и выберите <kbd>Удалённый</kbd> > <kbd>Добавить удалённый репозиторий</kbd>.

3. Выберите <kbd>Добавить удалённый репозиторий из GitHub</kbd>. При необходимости выполните любые этапы аутентификации, затем вернитесь в VS Code и повторите это действие.

4. Вы увидите список всех своих репозиториев на GitHub. Выберите тот, который вы создали для этого проекта. Если вы не видите свой проект, вставьте URL-адрес Github напрямую. Вам также могут попросить дать этому репозиторию локальное имя. Вы можете выбрать любое имя, которое пожелаете.

5. В верхней части меню будет место для ввода сообщения о **Сообщение** (описания изменений файла). Введите `initial commit` и нажмите кнопку <kbd>Фиксация</kbd>, чтобы зафиксировать эти изменения.

7. Вы можете увидеть сообщение, что у вас нет «проиндексированных» фиксаций, и вас попросят проиндексировать их. Щелкните <kbd>Всегда</kbd> и продолжайте.

8. Наконец, список измененных файлов должен быть заменен кнопкой <kbd>Опубликовать</kbd>. Нажмите на нее, чтобы отправить свои фиксации на Github.

### Проверьте свой проект GitHub

Чтобы убедиться, что ваш проект успешно сохранен на GitHub, перейдите на сайт GitHub.com и найдите на странице своего аккаунта список ваших репозиториев. Выберите новый репозиторий, который вы создали, и убедитесь, что он содержит файлы вашего проекта Astro.

우리는 모든 규모의 기여와 모든 기술 수준의 기여를 환영합니다. 오픈 소스 프로젝트로서 우리는 기여자에게 환원하는 것을 믿습니다. 우리는 PR, 기술 문서 작성 및 기능 아이디어를 현실로 전환하는 것에 대한 안내를 기꺼이 도와드립니다.

더 많은 참여를 원하시나요? 다양한 역할, 메인테이너 추천 프로세스, 코드 리뷰 프로세스 및 행동 강령 시행에 대한 자세한 설명은 [정책 문서](https://github.com/withastro/.github/blob/main/GOVERNANCE.md)를 참조하세요.

## 기여 방법

### 프로젝트 저장소

Astro 프로젝트에 기여할 수 있는 방법은 많습니다! 모든 Astro 저장소에는 프로젝트 루트에 존재하는 `CONTRIBUTING.md` 파일로 이동할 수 있는 링크가 포함된 README가 있습니다.

[Astro의 GitHub 프로필](https://github.com/withastro)을 방문하여 다음 저장소를 찾으세요.

- 공식 통합 및 시작 템플릿을 포함한 [기본 Astro 코드베이스](https://github.com/withastro/astro).

- 전체 Astro 웹사이트인 [Astro 문서](https://github.com/withastro/docs)! 작성된 콘텐츠뿐만 아니라 a11y, CSS, UI 및 UX 문제를 해결하는 Astro 코드에도 기여하세요. 또한 문서를 여러 언어로 제공하므로 전체 사이트를 번역하는 데 도움이 필요합니다.

- Go로 작성되었으며, WASM으로 배포되는 [Astro 컴파일러](https://github.com/withastro/compiler)

- Astro 언어(`.astro` 파일)에 필요한 편집기 도구인 Astro [언어 도구](https://github.com/withastro/language-tools)

- Astro의 공식 문서 프레임워크인 [Starlight](https://github.com/withastro/starlight)

- Astro의 미래를 그린 [Astro 로드맵](https://github.com/withastro/roadmap)! Astro 프로젝트에 대한 아이디어, 제안 및 공식 RFC 제안입니다.

### 기여 유형

자신의 코드나 콘텐츠를 제공하는 것 외에도 PR에 리뷰 댓글을 남기고, 기존 GitHub Issues 및 토론에 아이디어를 제공하고, "Pinned" issue 유지 관리 작업에 참여하여 큰 기여를 할 수도 있습니다!

모든 PR, 특히 번역 PR에는 리뷰어가 필요합니다! PR을 리뷰하고 댓글, 제안을 남기거나 "LGTM!"("Looks Good To Me!")과 함께 승인하는 것은 모든 저장소에서 기여를 시작하거나 Astro에 대해 더 많이 배우는 좋은 방법입니다.

또한 우리는 매우 활발한 [Discord](https://astro.build/chat) 커뮤니티를 보유하고 있습니다! 우리는 새로운 회원을 환영하고, 질문에 답하고, Astro를 위해 구축한 것을 공유하는 사람들의 기여를 소중히 여깁니다! 전통적인 GitHub 기여 외에도, Astro는 커뮤니티에 참여하고 블로그 게시물, 비디오 및 컨퍼런스 토크에서 Astro를 공유하고 커뮤니티의 건강을 유지하는 데 도움을 주는 커뮤니티 구성원을 인정하고 지원합니다.

## 문서에 기여하기

Astro 문서에 기여하는 데 도움이 되는 몇 가지 가이드가 있습니다.

오픈 소스에 처음으로 기여하거나 방금 구축한 새로운 Astro 기능에 대한 문서를 추가해야 하는 경우, 아니면 번역할 다음 페이지를 찾고 있는 숙련된 번역가인 경우, 혹은 PR 리뷰어로서 도움을 주는 방법에 대해 더 자세히 알고 싶은 경우... 우리가 도와드리겠습니다!

우리의 전용 웹사이트인 [*Astro Docs* Docs](https://contribute.docs.astro.build)를 방문하시면 여러분이 버그 수정, 문서 업데이트, 번역, 기능 추가 또는 PR 리뷰어로서 Astro 문서에 기여하는 데 도움을 드릴 수 있습니다.

क्या आप Astro इंस्टॉल करने के लिए तैयार हैं? आरंभ करने के लिए पहले `create-astro` CLI का उपयोग करने के लिए इस मार्गदर्शक का पालन करें।

:::note[क्या आप Astro को नियमावली रूप से इंस्टॉल करना पसंद करते हैं?]
इसके लिए हमारे [चरण-दर-चरण नियमावली इंस्टॉलेशन मार्गदर्शिका](/hi/install/manual/) को पढ़ें।
:::
#### आवश्यक निर्भरताएँ

- **Node.js** - `v18.14.1` या उससे उच्चतर।
- **टेक्स्ट एडीटर** - हम अपने [आधिकारिक Astro एक्सटेंशन](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode) के साथ [VS Code](https://code.visualstudio.com/) की अनुशंसा करते हैं।
- **टर्मिनल** - Astro का उपयोग उसके कमांड-लाइन इंटरफ़ेस(CLI) के माध्यम से किया जाता है।

#### इंस्टॉलेशन

`create astro` किसी नए Astro परियोजना को नए सिरे से शुरू करने का सबसे तेज़ तरीका है। यह आपके लिए एक नयी Astro परियोजना को इंस्टॉल करने के हर चरण में आपका मार्गदर्शन करेगा। यह आपको कुछ अलग-अलग आधिकारिक प्रारंभक टेम्पलेट्स में से चुनने की अनुमति देता है या आप [GitHub पर --template तर्क के साथ किसी भी मौजूदा परियोजना का उपयोग भी कर सकते हैं](#प्रारंभक-टेम्पलेट्स)।

:::tip[ऑनलाइन पूर्वावलोकन]
क्या आप अपने ब्राउज़र में Astro आज़माना पसंद करेंगे? हमारे स्टार्टर टेम्प्लेट ब्राउज़ करने और अपना ब्राउज़र छोड़े बिना एक नयी Astro परियोजना तैयार करने के लिए [astro.new](https://astro.new/) पर जाएँ।
:::

## 1. सेटअप विज़ार्ड चलाएँ

हमारे सुविधाजनक इंस्टॉल विज़ार्ड को शुरू करने के लिए अपने टर्मिनल में निम्नलिखित कमांड चलाएँ:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # npm के साथ एक नयी परियोजना बनाएं
  npm create astro@latest
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # pnpm के साथ एक नयी परियोजना बनाएं
  pnpm create astro@latest
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # yarn के साथ एक नयी परियोजना बनाएं
  yarn create astro
  ```
  </Fragment>
</PackageManagerTabs>

आप अपने उपकरण पर कहीं भी `create astro` चला सकते हैं, इसलिए शुरू करने से पहले अपने परियोजना के लिए एक नई खाली निर्देशिका बनाने की कोई आवश्यकता नहीं है। यदि आपके पास अपने नए परियोजना के लिए अभी तक कोई खाली निर्देशिका नहीं है, तो विज़ार्ड स्वचालित रूप से आपके लिए एक निर्देशिका बनाने में मदद करेगा।

यदि सब कुछ ठीक रहा, तो आपको "Liftoff confirmed. Explore your project!" दिखना चाहिए जिसके बाद कुछ अनुशंसित कदम उठाए जाएंगे। Astro का उपयोग शुरू करने के लिए अपनी नई परियोजना निर्देशिका में प्रवेश करे `cd` कमांड का उपयोग करके ।

यदि आपने CLI विज़ार्ड के दौरान `npm install` चरण को छोड़ दिया है, तो जारी रखने से पहले अपनी निर्भरताएं इंस्टॉल करना सुनिश्चित करें।

## 2. Astro प्रारंभ करें ✨

Astro एक अंतर्निर्मित विकास सर्वर के साथ आता है जिसमें परियोजना विकास के लिए आपकी ज़रूरत की हर चीज़ मौजूद है। `astro dev` कमांड स्थानीय विकास सर्वर शुरू करेगा ताकि आप अपनी नई वेबसाइट को पहली बार काम करते हुए देख सकें।

प्रत्येक स्टार्टर टेम्प्लेट एक पूर्व-कॉन्फ़िगर्ड स्क्रिप्ट के साथ आता है जो आपके लिए `astro dev` चलाएगा। इस कमांड को चलाने और Astro विकास सर्वर शुरू करने के लिए अपने पसंदीदा पैकेज मैनेजर का उपयोग करें।

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npm run dev
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm run dev
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn run dev
  ```
  </Fragment>
</PackageManagerTabs>


यदि सब कुछ ठीक रहा, तो Astro को अब आपके परियोजना को [http://localhost:4321/](http://localhost:4321/) पर सेवा देनी चाहिए!

Astro आपकी `src/` निर्देशिका में लाइव फ़ाइल परिवर्तनों का जाँच करेगा, इसलिए जब आप विकास के दौरान परिवर्तन करेंगे तो आपको सर्वर को पुनरारंभ करने की आवश्यकता नहीं होगी।

यदि आप ब्राउज़र में अपनी परियोजना खोलने में परेशानी महसूस कर रहे हैं, तो उस टर्मिनल पर वापस जाएँ जहाँ आपने `dev` कमांड चलाया था और देखें कि क्या कोई त्रुटि हुई है, या क्या आपकी परियोजना एक से भिन्न URL पर परोसा जा रहा है ऊपर से जुड़ा हुआ.

## प्रारंभक टेम्पलेट्स

आप `create astro` कमांड में --template तर्क डालके किसी [आधिकारिक उदाहरण](https://github.com/withastro/astro/tree/main/examples) या किसी GitHub रिपॉजिटरी की `main` शाखा के आधार पर एक नयी Astro परियोजना भी शुरू कर सकते हैं।

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # आधिकारिक उदाहरण के साथ एक नयी परियोजना बनाएं
  npm create astro@latest -- --template <example-name>

  # GitHub रिपॉजिटरी की मुख्य शाखा के आधार पर एक नयी परियोजना बनाएं
  npm create astro@latest -- --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # आधिकारिक उदाहरण के साथ एक नयी परियोजना बनाएं
  pnpm create astro@latest --template <example-name>
  
  # GitHub रिपॉजिटरी की मुख्य शाखा के आधार पर एक नयी परियोजना बनाएं
  pnpm create astro@latest --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # आधिकारिक उदाहरण के साथ एक नयी परियोजना बनाएं
  yarn create astro --template <example-name>
  
  # GitHub रिपॉजिटरी की मुख्य शाखा के आधार पर एक नयी परियोजना बनाएं
  yarn create astro --template <github-username>/<github-repo>
  ```
  </Fragment>
</PackageManagerTabs>

डिफ़ॉल्ट रूप से, यह कमांड एक टेम्पलेट रिपॉजिटरी की मुख्य शाखा का उपयोग करेगा। किसी भिन्न शाखा का उपयोग करने के लिए, इसे इस रूप में : `--template` argument: `<github-username>/<github-repo>#<branch>` उपयोग करे।

हमारे [थीम और स्टार्टर्स प्रदर्शन](https://astro.build/themes/) का अन्वेषण करें जहां आप ब्लॉग, पोर्टफ़ोलियो, दस्तावेज़ीकरण, लैंडिंग पेज और बहुत कुछ के लिए थीम ब्राउज़ कर सकते हैं! या, और भी अधिक स्टार्टर परियोजनाओं के लिए [GitHub पर खोजें](https://github.com/search?o=desc&q=astro+starter&s=stars&type=Repositories)।

यहां कुछ विषय दिए गए हैं जिन्हें हम आगे अन्वेषण करने की सलाह देते हैं। आप इन्हें किसी भी क्रम में पढ़ सकते हैं. आप हमारे दस्तावेज़ीकरण को थोड़ी देर के लिए छोड़ भी सकते हैं और अपने नए Astro परियोजना के कोडबेस में खेल सकते हैं, जब भी आप किसी परेशानी में पड़ें या कोई प्रश्न हो तो यहां वापस आएं।

📚 **अपने कोडबेस को समझें:** हमारे [परियोजना संरचना मार्गदर्शिका](/hi/core-concepts/project-structure/) में Astro के परियोजना की संरचना के बारे में और जानें।

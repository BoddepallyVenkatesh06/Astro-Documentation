Astro utilizza il **routing basato su file** per generare gli URL di build in base al layout del file della directory `src/pages/` del tuo progetto.

## Navigazione tra le pagine

Astro utilizza gli [elementi HTML standard `<a>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/a) per navigare tra i percorsi. Non viene fornito alcun componente `<Link>` specifico del framework.


## Percorsi statici

[I componenti della pagina](/it/core-concepts/astro-pages/) `.astro` nonch√© file Markdown e MDX (`.md`, `.mdx`) nella directory `src/pages/` **diventano automaticamente pagine del tuo sito web**. Il percorso di ogni pagina corrisponde al suo percorso e al nome file all'interno della directory `src/pages/`.

```diff
# Esempio: percorsi statici

src/pages/about.astro        -> mysite.com/about
src/pages/about/index.astro  -> mysite.com/about
```

:::tip

Nelle build statiche, puoi personalizzare il formato di output del file utilizzando l'opzione di configurazione [`build.format`](/it/reference/configuration-reference/#buildformat).

## Ordine di priorit√† del percorso

√à possibile che pi√π percorsi corrispondano allo stesso percorso URL. Ad esempio, ciascuno di questi percorsi corrisponderebbe a `/posts/create`:

- src/pages/

    - [pid].astro

Astro ha bisogno di sapere quale percorso deve essere utilizzato per creare la pagina. Per fare ci√≤, li ordina secondo le seguenti regole:

- I percorsi statici senza parametri di percorso avranno la precedenza su tutti gli altri percorsi

- I percorsi dinamici pre-renderizzati hanno la precedenza sui percorsi dinamici del server
- I parametri rest hanno la priorit√† pi√π bassa
- Gli endpoint hanno sempre la precedenza sulle pagine
- I pareggi vengono risolti in ordine alfabetico

Considerando l'esempio sopra, ecco alcuni esempi di come le regole abbineranno un URL richiesto al percorso utilizzato per creare l'HTML:

- `pages/posts/create.astro` - Costruir√† `/posts/create`
- `pages/posts/[pid].astro` - Costruir√† `/posts/1`, `/posts/abc`, ecc. Ma non `/posts/create`
- `pages/posts/[...slug].astro` - Costruir√† `/posts/1/2`, `/posts/a/b/c`, ecc. Ma non `/posts/create`, `/post/1`, `/post/abc`

Anche i reindirizzamenti seguono le stesse regole, ma hanno la priorit√† *ultima*; se sono presenti un percorso basato su file e un reindirizzamento con lo stesso livello di priorit√† del percorso, viene scelto il percorso basato su file.

### Impaginazione all'interno di altri percorsi

Un caso d'uso pi√π avanzato per l'impaginazione √® l'**impaginazione all'interno di altri percorsi.** Questo avviene quando l'impaginazione viene combinata con altri parametri di percorso dinamici.
